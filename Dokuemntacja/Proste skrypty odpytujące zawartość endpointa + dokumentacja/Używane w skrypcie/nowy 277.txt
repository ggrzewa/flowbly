# DataForSEO Google Trends Explore API - Szybka Dokumentacja

## PODSTAWOWE INFO
- Endpoint: /v3/keywords_data/google_trends/explore/live
- Method: POST (Live - instant results)
- Format: JSON (UTF-8)
- Limit: 2000 calls/min, max 250 Live tasks/min
- Max keywords: 5 per request

---

## POLA ŻĄDANIA (POST REQUEST)

| Pole | Typ | Wymagane | Opis |
|------|-----|----------|------|
| keywords | array | TAK | Słowa kluczowe (max 5, max 100 znaków/słowo) |
| location_name | string | NIE | Pełna nazwa lokalizacji (np. "United States") |
| location_code | integer | NIE | Kod lokalizacji (np. 2840) |
| language_name | string | NIE | Pełna nazwa języka (default: English) |
| language_code | string | NIE | Kod języka (default: en) |
| type | string | NIE | Typ: web/news/youtube/images/froogle (default: web) |
| category_code | integer | NIE | Kategoria wyszukiwania (default: 0 = wszystkie) |
| date_from | string | NIE | Data od (yyyy-mm-dd), min: 2004-01-01 dla web |
| date_to | string | NIE | Data do (yyyy-mm-dd) |
| time_range | string | NIE | past_hour/past_4_hours/past_day/past_7_days/past_30_days/past_90_days/past_12_months/past_5_years/2004_present/2008_present |
| item_types | array | NIE | Typy elementów: google_trends_graph/google_trends_map/google_trends_topics_list/google_trends_queries_list |
| tag | string | NIE | Identyfikator zadania (max 255 znaków) |

---

## GŁÓWNA STRUKTURA ODPOWIEDZI

```
response
├── version (string)
├── status_code (integer)
├── status_message (string)
├── time (string)
├── cost (float)
├── tasks_count (integer)
├── tasks_error (integer)
└── tasks (array)
    └── [task_object]
```

## OBIEKT ZADANIA (tasks[])

| Pole | Typ | Opis |
|------|-----|------|
| id | string | UUID zadania |

| status_message | string | Wiadomość statusu |
| time | string | Czas wykonania |
| cost | float | Koszt w USD |
| result_count | integer | Liczba wyników |
| path | array | Ścieżka URL |
| data | object | Parametry z POST request |

## STRUKTURA WYNIKU (result[])

### PODSTAWOWE POLA WYNIKU
| Pole | Typ | Opis |
|------|-----|------|
| keywords | array | Słowa kluczowe z POST |
| location_code | integer | Kod lokalizacji (null jeśli brak) |
| language_code | string | Kod języka (null jeśli brak) |
| check_url | string | Bezpośredni URL do Google Trends |
| datetime | string | Data UTC: yyyy-mm-dd hh-mm-ss +00:00 |
| items_count | integer | Liczba elementów w items |
| items | array | Główne dane wyników |

## ELEMENTY W ITEMS[]

### 1. GOOGLE_TRENDS_GRAPH
```
google_trends_graph
├── position (integer) - pozycja elementu
├── type (string) - "google_trends_graph"
├── title (string) - tytuł elementu
├── keywords (array) - słowa kluczowe
├── data (array)
│   ├── date_from (string) - data od
│   ├── date_to (string) - data do
│   ├── timestamp (integer) - Unix timestamp
│   ├── missing_data (boolean) - czy brak danych
│   └── values (array) - wartości popularności (0-100/null)
└── averages (array) - średnie wartości
```

### 2. GOOGLE_TRENDS_MAP
```
google_trends_map
├── position (integer)
├── type (string) - "google_trends_map"
├── title (string) - tytuł elementu
├── keywords (array)
└── data (array)
    ├── geo_id (string) - ID lokalizacji (np. US-NY)
    ├── geo_name (string) - nazwa lokalizacji
    ├── values (array) - wartości popularności (0-100/null)
    └── max_value_index (integer) - indeks max wartości
```

### 3. GOOGLE_TRENDS_TOPICS_LIST
```
google_trends_topics_list
├── position (integer)
├── type (string) - "google_trends_topics_list"
├── title (string) - tytuł elementu
├── keywords (array)
└── data (object)
    ├── top (array) - najpopularniejsze tematy
    │   ├── topic_id (string) - unikalny ID tematu
    │   ├── topic_title (string) - tytuł tematu
    │   ├── topic_type (string) - typ tematu
    │   └── value (integer) - popularność (0-100)
    └── rising (array) - rosnące tematy
        ├── topic_id (string)
        ├── topic_title (string)
        ├── topic_type (string)
        └── value (integer) - wzrost w % (bez znaku %)
```

### 4. GOOGLE_TRENDS_QUERIES_LIST
```
google_trends_queries_list
├── position (integer)
├── type (string) - "google_trends_queries_list"
├── title (string) - tytuł elementu
├── keywords (array)
└── data (object)
    ├── top (array) - najpopularniejsze zapytania
    │   ├── query (string) - zapytanie
    │   └── value (integer) - popularność (0-100)
    └── rising (array) - rosnące zapytania
        ├── query (string) - zapytanie
        └── value (integer) - popularność (0-100)
```



## WAŻNE NOTATKI

**LIMITY I OGRANICZENIA:**
- Live method: max 250 zadań/min (vs Standard 2000/min)
- Keywords: max 5, max 100 znaków każde, min 2 znaki
- Zabronione znaki w keywords: < > | \ " - + = ~ ! : * ( ) [ ] { }
- topics_list i queries_list: tylko przy 1 słowie kluczowym
- Przecinki w keywords są usuwane

**WARTOŚCI:**
- values: 100 = szczyt, 50 = połowa, 0 = brak danych, null = brak danych
- missing_data: true = brak danych (kropkowana linia w Google Trends)
- rising values: wzrost w % (bez znaku %)

**DATY:**
- Minimalne daty: web (2004-01-01), inne typy (2008-01-01)
- time_range vs date_from/date_to: time_range ignorowane gdy są daty
- Format daty: yyyy-mm-dd

**LOKALIZACJE I JĘZYKI:**
- Sprawdź dostępne przez /locations i /languages endpoints
- location_name/location_code - użyj jedno
- language_name/language_code - użyj jedno

**INNE:**
- check_url: bezpośredni link do Google Trends dla weryfikacji
- Koszt naliczany za request, nie za liczbę słów
- item_types: określ konkretne typy dla szybszego wykonania

import os
import traceback
from fastapi import APIRouter, HTTPException
from pydantic import BaseModel
from dotenv import load_dotenv
import requests
from requests.auth import HTTPBasicAuth

load_dotenv()

router = APIRouter()

DFS_LOGIN = os.getenv("DATAFORSEO_LOGIN")
DFS_PASSWORD = os.getenv("DATAFORSEO_PASSWORD")

class GoogleTrendsInput(BaseModel):
    keywords: list[str]

@router.post("/trends/google")
async def get_google_trends(data: GoogleTrendsInput):
    if not all([DFS_LOGIN, DFS_PASSWORD]):
        raise HTTPException(status_code=500, detail="Brak danych logowania do API DataForSEO.")

    if not data.keywords:
        raise HTTPException(status_code=422, detail="Musisz podać przynajmniej jedno słowo kluczowe.")

    url = "https://api.dataforseo.com/v3/keywords_data/google_trends/explore/live"

    payload = [
        {
            "keywords": data.keywords,
            "location_name": "Poland",
            "language_code": "pl",
            "type": "web",
            "item_types": [
                "google_trends_graph",
                "google_trends_map",
                "google_trends_topics_list",
                "google_trends_queries_list"
            ]
        }
    ]

    try:
        response = requests.post(
            url,
            auth=HTTPBasicAuth(DFS_LOGIN, DFS_PASSWORD),
            json=payload
        )

        if response.status_code != 200:
            raise HTTPException(status_code=500, detail=f"Błąd API DFS: {response.status_code} – {response.text}")

        json_data = response.json()
        tasks = json_data.get("tasks", [])
        if not tasks or not tasks[0].get("result"):
            raise HTTPException(status_code=404, detail="Brak wyników dla podanych słów kluczowych.")

        return tasks[0]["result"][0]  # pełna zawartość `result`

    except Exception as e:
        traceback_str = traceback.format_exc()
        print("❌ Błąd przy zapytaniu HTTP do DataForSEO:")
        print(traceback_str)
        raise HTTPException(status_code=500, detail=f"Błąd serwera:\n{str(e)}")
