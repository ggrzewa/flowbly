# DataForSEO Trends Merged Data API - Szybka Dokumentacja

## PODSTAWOWE INFO
- Method: POST
- Format: JSON (UTF-8)
- Limit: 2000 calls/min, max 30 simultaneous
- Max keywords: 5 per request

---

## POLA ŻĄDANIA (POST REQUEST)

| Pole | Typ | Wymagane | Opis |
|------|-----|----------|------|
| keywords | array | TAK | Słowa kluczowe (max 5) |
| location_name | string | NIE | Pełna nazwa lokalizacji (np. "United Kingdom") |
| location_code | integer | NIE | Kod lokalizacji (np. 2840) |
| type | string | NIE | Typ: web/news/ecommerce (default: web) |
| date_from | string | NIE | Data od (yyyy-mm-dd), min: 2004-01-01 dla web |
| date_to | string | NIE | Data do (yyyy-mm-dd) |
| time_range | string | NIE | past_4_hours/past_day/past_7_days/past_30_days/past_90_days/past_12_months/past_5_years |
| tag | string | NIE | Identyfikator zadania (max 255 znaków) |

---

## GŁÓWNA STRUKTURA ODPOWIEDZI

```
response
├── version (string)
├── status_code (integer)
├── status_message (string)
├── time (string)
├── cost (float)
├── tasks_count (integer)
├── tasks_error (integer)
└── tasks (array)
    └── [task_object]
```

---

## OBIEKT ZADANIA (tasks[])

| Pole | Typ | Opis |
|------|-----|------|
| id | string | UUID zadania |
| status_code | integer | Status kod (10000-60000) |
| status_message | string | Wiadomość statusu |
| time | string | Czas wykonania |
| cost | float | Koszt w USD |
| result_count | integer | Liczba wyników |
| path | array | Ścieżka URL |
| data | object | Parametry z POST request |
| result | array | Tablica wyników |

---

## STRUKTURA WYNIKU (result[])

### PODSTAWOWE POLA WYNIKU
| Pole | Typ | Opis |
|------|-----|------|
| keywords | array | Słowa kluczowe z POST |
| type | array | Typ wyszukiwarki z POST |
| location_code | integer | Kod lokalizacji (null jeśli brak) |
| language_code | string | Kod języka (null jeśli brak) |
| datetime | string | Data UTC: yyyy-mm-dd hh-mm-ss +00:00 |
| items_count | integer | Liczba elementów w items |
| items | array | Główne dane wyników |

---

## ELEMENTY W ITEMS[]

### 1. DATAFORSEO_TRENDS_GRAPH
```
dataforseo_trends_graph
├── position (integer) - pozycja elementu
├── type (string) - "dataforseo_trends_graph"
├── keywords (array) - słowa kluczowe
├── data (array)
│   ├── date_from (string) - data od
│   ├── date_to (string) - data do
│   ├── timestamp (integer) - Unix timestamp
│   └── values (array) - wartości popularności (0-100)
└── averages (array) - średnie wartości
```

### 2. SUBREGION_INTERESTS
```
subregion_interests
├── position (integer)
├── type (string) - "subregion_interests"
├── keywords (array)
├── interests (array)
│   ├── keyword (string)
│   └── values (array)
│       ├── geo_id (string) - ID lokalizacji (np. US-NY)
│       ├── geo_name (string) - nazwa lokalizacji
│       └── value (integer) - popularność (0-100)
└── interests_comparison (object/null)
    ├── items (array)
    │   ├── geo_id (string)
    │   ├── geo_name (string)
    │   └── values (array) - wartości dla każdego słowa
    └── absolute_items (array)
        ├── geo_id (string)
        ├── geo_name (string)
        └── values (array) - wartości absolutne
```

### 3. DEMOGRAPHY
```
demography
├── position (integer)
├── type (string) - "demography"
├── keywords (array)
├── demography (object)
│   ├── age (array)
│   │   ├── keyword (string)
│   │   └── values (array)
│   │       ├── type (string) - "18-24"/"25-34"/"35-44"/"45-54"/"55-64"
│   │       └── value (integer) - popularność (0-100)
│   └── gender (array)
│       ├── keyword (string)
│       └── values (array)
│           ├── type (string) - "female"/"male"
│           └── value (integer) - popularność (0-100)
└── demography_comparison (object/null)
    ├── age (object)
    │   └── $18-24, $25-34, $35-44, $45-54, $55-64 (array)
    └── gender (object)
        ├── female (array)
        └── male (array)
```

---


## WAŻNE NOTATKI

- Wartości popularności: 100 = szczyt, 50 = połowa, 0 = brak danych
- interests_comparison i demography_comparison = null przy 1 słowie kluczowym
- Minimalne daty: web (2004-01-01), inne typy (2008-01-01)
- Lokalizacje: sprawdź dostępne przez /locations endpoint
- Koszt naliczany za request, nie za liczbę słów kluczowych

import os
from fastapi import APIRouter, HTTPException
from pydantic import BaseModel
from dotenv import load_dotenv
from dataforseo_client import configuration as dfs_config, api_client as dfs_api_provider
from dataforseo_client.api.keywords_data_api import KeywordsDataApi
from dataforseo_client.models.keywords_data_dataforseo_trends_merged_data_live_request_info import KeywordsDataDataforseoTrendsMergedDataLiveRequestInfo

load_dotenv()

router = APIRouter()

DFS_LOGIN = os.getenv("DATAFORSEO_LOGIN")
DFS_PASSWORD = os.getenv("DATAFORSEO_PASSWORD")

# Input model
class TrendsMergedInput(BaseModel):
    keywords: list[str]

@router.post("/trends/merged")
async def get_trends_merged(data: TrendsMergedInput):
    if not all([DFS_LOGIN, DFS_PASSWORD]):
        raise HTTPException(status_code=500, detail="Brak danych logowania do API DataForSEO.")

    config = dfs_config.Configuration(username=DFS_LOGIN, password=DFS_PASSWORD)

    request_data = [
        KeywordsDataDataforseoTrendsMergedDataLiveRequestInfo(
            keywords=data.keywords,
            location_code=2616,      # Polska
            language_code="pl",      # Polski
            type="web"               # Zakres danych: wyszukiwarka Google
        )
    ]

    with dfs_api_provider.ApiClient(config) as client:
        api_instance = KeywordsDataApi(client)

        try:
            api_response = api_instance.dataforseo_trends_merged_data_live(request_data)
            task = api_response.tasks[0]
            if not task.result:
                raise HTTPException(status_code=404, detail="Brak wyników dla podanych słów kluczowych.")
            return task.result[0].to_dict()
        except Exception as e:
            raise HTTPException(status_code=500, detail=str(e))
