# DataForSEO Related Keywords API - Szybka Dokumentacja

## PODSTAWOWE INFO
- Endpoint: /v3/dataforseo_labs/google/related_keywords/live
- Method: POST (Live - instant results)
- Format: JSON (UTF-8)
- Limit: 2000 calls/min, max 30 simultaneous
- Max results: 4680 keywords (depth 4)
- Źródło: "searches related to" SERP element

---

## ALGORYTM WYSZUKIWANIA
- **Metoda**: Depth-first search
- **Bazuje na**: "searches related to" z SERP
- **Przykład**: "keyword research" → "free keyword research", "keyword research tools", etc.

---

## POLA ŻĄDANIA (POST REQUEST)

| Pole | Typ | Wymagane | Opis |
|------|-----|----------|------|
| keyword | string | TAK | Główne słowo kluczowe (UTF-8, lowercase) |
| location_name | string | TAK* | Pełna nazwa lokalizacji |
| location_code | integer | TAK* | Kod lokalizacji |
| language_name | string | TAK* | Pełna nazwa języka |
| language_code | string | TAK* | Kod języka |
| depth | integer | NIE | Głębokość wyszukiwania (0-4, default: 1) |
| include_seed_keyword | boolean | NIE | Dołączyć dane seed keyword (default: false) |
| include_serp_info | boolean | NIE | Dołączyć dane SERP (default: false) |
| include_clickstream_data | boolean | NIE | Dołączyć clickstream (default: false, 2x koszt) |
| ignore_synonyms | boolean | NIE | Ignorować synonimy (default: false) |
| replace_with_core_keyword | boolean | NIE | Zwracać dane dla core keyword (default: false) |
| filters | array | NIE | Filtry wyników (max 8) |
| order_by | array | NIE | Sortowanie (max 3 reguły) |
| limit | integer | NIE | Max wyników (default: 100, max: 1000) |
| offset | integer | NIE | Offset w wynikach (default: 0) |
| tag | string | NIE | Identyfikator zadania (max 255 znaków) |

*wymagane po jednym z pary: location_name/location_code, language_name/language_code

 "keyword_data.keyword_info.cpc,desc"]

**COST OPTIMIZATION:**
- **include_clickstream_data: true** = 2x koszt
- Używaj tylko gdy potrzebujesz demografii
- **include_serp_info** zwiększa response size
- **depth 4** może generować ogromne koszty

**QUALITY CONTROL:**
- **replace_with_core_keyword** dla consistent data
- **ignore_synonyms** eliminuje duplikaty
- Sprawdzaj **detected_language** vs **is_another_language**
- Analizuj **keyword_difficulty** dla feasibility

**USE CASES:**
- **Seed keyword expansion** (depth 1-2)

import os
import logging
from fastapi import APIRouter, HTTPException
from pydantic import BaseModel
from dotenv import load_dotenv
from dataforseo_client import configuration as dfs_config, api_client as dfs_api_provider
from dataforseo_client.api.dataforseo_labs_api import DataforseoLabsApi
from dataforseo_client.models.dataforseo_labs_google_related_keywords_live_request_info import (
    DataforseoLabsGoogleRelatedKeywordsLiveRequestInfo,
)

load_dotenv()

router = APIRouter()

# Konfiguracja logowania
logger = logging.getLogger("dfs_labs")
logger.setLevel(logging.DEBUG)
handler = logging.StreamHandler()
formatter = logging.Formatter("%(asctime)s | %(levelname)s | %(message)s")
handler.setFormatter(formatter)
logger.addHandler(handler)

# Dane logowania
DFS_LOGIN = os.getenv("DATAFORSEO_LOGIN")
DFS_PASSWORD = os.getenv("DATAFORSEO_PASSWORD")

# Model danych wejściowych
class RelatedKeywordsInput(BaseModel):
    keyword: str
    location_code: int = 2616  # domyślnie Polska
    language_code: str = "pl"

@router.post("/labs/related-keywords")
async def get_related_keywords(data: RelatedKeywordsInput):
    if not DFS_LOGIN or not DFS_PASSWORD:
        logger.error("❌ Brak danych logowania do API")
        raise HTTPException(status_code=500, detail="Brak danych logowania do API DataForSEO.")

    logger.info(f"➡️ Otrzymano zapytanie dla słowa kluczowego: {data.keyword}")

    config = dfs_config.Configuration(username=DFS_LOGIN, password=DFS_PASSWORD)

    request_data = [
        DataforseoLabsGoogleRelatedKeywordsLiveRequestInfo(
            keyword=data.keyword,
            location_code=data.location_code,
            language_code=data.language_code
        )
    ]

    try:
        with dfs_api_provider.ApiClient(config) as api_client:
            api_instance = DataforseoLabsApi(api_client)

            logger.debug("➡️ Wysyłanie żądania do DataForSEO Labs API (Related Keywords)...")
            api_response = api_instance.google_related_keywords_live(request_data)

            task = api_response.tasks[0]
            if not task.result:
                logger.warning("⚠️ Brak wyników dla tego słowa kluczowego.")
                raise HTTPException(status_code=404, detail="Brak danych dla podanego słowa kluczowego.")

            logger.info("✅ Pobrano dane z DFS Labs API (Related Keywords).")
            return task.result[0].to_dict()

    except Exception as e:
        logger.exception("❌ Błąd podczas pobierania danych z DataForSEO Labs API")
        raise HTTPException(status_code=500, detail=f"Błąd serwera:\n{str(e)}")
